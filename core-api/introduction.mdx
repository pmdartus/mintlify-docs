The Nabla Core API turns raw clinical audio into structured, EHR-ready data. It powers medical transcription, clinical note generation, FHIR-normalized data extraction, dictation, and Dot Phrase suggestions — all tuned for healthcare workflows.

Whether you're building a telehealth platform, an EHR, or a voice-enabled documentation tool, the Core API gives you a small set of composable endpoints you can wire into existing systems quickly.

<Tip>
**New here?** Head straight to the [Quickstart](/core-api/quickstart) to make your first call in under five minutes, then pick a Guide for the workflow you're building.
</Tip>

## What you can build

Each capability has a dedicated guide with runnable code and "next steps" links.

| Capability | What it does | Guide |
|---|---|---|
| Transcription (real-time) | Stream audio over WebSocket, receive transcript items as they arrive. | [Stream a live encounter](/core-api/guides/stream-a-live-encounter) |
| Transcription (recorded) | Send an audio file (≤10 min sync, ≤60 min async) and get a full transcript back. | [Transcribe a recorded encounter](/core-api/guides/transcribe-a-recorded-encounter) |
| Clinical note generation | Turn a transcript into a structured note using SOAP, APSO, WCC, or specialty templates. | [Generate a clinical note](/core-api/guides/generate-a-clinical-note) |
| Magic Edit | Refine a generated note with a natural-language instruction (e.g., "Make it more concise"). | [Edit a note with Magic Edit](/core-api/guides/edit-a-note-with-magic-edit) |
| Patient-friendly instructions | Generate post-visit summaries tuned for the patient or a caregiver. | [Generate patient instructions](/core-api/guides/generate-patient-instructions) |
| FHIR-normalized data | Extract conditions (ICD-10) and observations (LOINC) from a generated note. | [Extract FHIR-normalized data](/core-api/guides/extract-fhir-normalized-data) |
| Dictation | Real-time or async dictation with explicit punctuation commands. | [Use dictation commands](/core-api/reference/dictation-commands) |
| Dot Phrases & custom dictionary | Provider-defined macros and term lists that improve recognition and structure. | [Dot Phrases & custom dictionaries](/core-api/guides/dot-phrases-and-custom-dictionaries) |
| EHR integration | Push Nabla output into Epic, Cerner, or any web-based EHR via the browser or your backend. | [Integrate with an EHR](/core-api/guides/ehr-integration) |

## Two surfaces: Server API and User API

The Core API exposes two surfaces with different auth shapes:

- **Server API** — backend-to-backend, OAuth 2.0 client credentials with a JWT bearer assertion.
- **User API** — short-lived per-user tokens issued by your backend, used directly from a frontend or mobile client.

If you're not sure which to use, start with the [Server vs User API](/core-api/concepts/server-vs-user-api) concept page.
