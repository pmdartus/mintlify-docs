title: "POST /encounters/url"
description: "Generate a fresh one-time launch URL for an existing Nabla encounter."
icon: "code"
---

Generate a fresh one-time URL for an encounter that has already been created. Use this when the provider closed the Nabla tab, the previous URL expired, or you need to re-open the same encounter without re-sending its context.

Nabla matches the user corresponding to your provider and the encounter, then returns a one-time URL your frontend navigates to. Navigating to it logs the provider into Nabla automatically and opens the app on this encounter.

<Note>
`POST /encounter/url` (singular) is a **deprecated alias** with identical behavior. New integrations should call `POST /encounters/url`.
</Note>

## Endpoint

```
POST https://<region>.api.nabla.com/v1/connect/server/encounters/url
```

`<region>` is `us` or `eu` — use the region your organization was provisioned in.

## Authentication

Authenticated with a **server access token** obtained from the OAuth 2.0 client credentials flow at `POST /oauth/token`. See [Authentication](/connect/authentication).

## Headers

| Header | Required | Description |
|---|---|---|
| `Authorization` | yes | `Bearer <access_token>` from the OAuth client credentials flow. |
| `X-Nabla-Api-Version` | recommended | Pins the API version (e.g., `2026-04-24`). See [Targeting a specific version](/connect/api-versioning/targeting-a-version). |
| `Content-Type` | yes | `application/json` |

## Request body

| Field | Type | Required | Description |
|---|---|---|---|
| `external_provider_id` | string | yes | Stable provider identifier from your EHR. Must match the provider who owns the encounter. |
| `external_encounter_id` | string | yes | Stable encounter identifier from your EHR. Must reference an existing encounter created via [`POST /encounters`](/connect/reference/server-api/create-encounter). |

### Example request body

```json
{
  "external_provider_id": "prov_8421",
  "external_encounter_id": "enc_2026_05_12_001"
}
```

### Example request

<Tabs>
<Tab title="cURL">

```bash
curl -X POST https://us.api.nabla.com/v1/connect/server/encounters/url \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "X-Nabla-Api-Version: 2026-04-24" \
  -H "Content-Type: application/json" \
  -d '{
    "external_provider_id": "prov_8421",
    "external_encounter_id": "enc_2026_05_12_001"
  }'
```

</Tab>
<Tab title="Node">

```js
const res = await fetch(
  "https://us.api.nabla.com/v1/connect/server/encounters/url",

      "X-Nabla-Api-Version": "2026-04-24",

    },
    body: JSON.stringify({

    }),
  },

const { encounter_url } = await res.json();
```

## Response

```json
{ "encounter_url": "https://app.nabla.com/connect/launch/<one-time-token>" }
```

| Field | Type | Description |
|---|---|---|
| `encounter_url` | string | One-time URL the frontend navigates to. Logs the provider in automatically and opens the app on this encounter. |

## Behavior notes

<Warning>
`encounter_url` is **one-time use** and expires **10 minutes** after creation. Don't store it, don't cache it, and don't pass it through long-lived queues. Mint a new one on demand each time you need to navigate the provider back into the encounter.
</Warning>
