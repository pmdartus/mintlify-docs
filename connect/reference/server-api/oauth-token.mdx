title: "Create server access token"
description: "Exchange a signed JWT client assertion for a Connect Server API access token."
api: "POST /v1/connect/server/oauth/token"
authMethod: "none"
---

Exchange a signed JWT client assertion for a short-lived server access token. This is the bootstrap endpoint of the OAuth 2.0 client credentials flow used by every other Connect Server API call.

<Note>
This page documents the endpoint contract only. For the full OAuth client setup — creating the client, configuring a JWKS URL or static public key, and constructing the JWT client assertion — see [Authentication](/connect/authentication).
</Note>

The token URL **must** match the `aud` claim in your JWT client assertion exactly. The playground above defaults to the `us` region; switch the server URL to `https://eu.api.nabla.com` if your organization was provisioned in EU.

## Body

<ParamField body="grant_type" type="string" required default="client_credentials">
  Must be `client_credentials`.
</ParamField>

<ParamField body="client_assertion_type" type="string" required default="urn:ietf:params:oauth:client-assertion-type:jwt-bearer">
  Must be `urn:ietf:params:oauth:client-assertion-type:jwt-bearer`.
</ParamField>

<ParamField body="client_assertion" type="string" required>
  The signed JWT client assertion. See [Construct a JWT client assertion](/connect/authentication#2-construct-a-jwt-client-assertion) for the required claims and signing.
</ParamField>

## Response

<ResponseField name="access_token" type="string" required>
  The server access token. Pass it as `Authorization: Bearer <access_token>` on every other Connect Server API request.
</ResponseField>

<ResponseField name="expires_in" type="integer" required>
  Token TTL in seconds. Typically `3600` (one hour) but may change without notice — always trust `expires_in` over a hard-coded value.
</ResponseField>

<Tip>
Cache the token and reuse it across requests; mint a new one only when it nears expiry. Repeating the exchange on every call is unnecessary and rate-limited.
</Tip>

## Errors

Errors follow the OAuth 2.0 error response format: a JSON body with an `error` code (and often an `error_description`).

| HTTP | `error` | Likely cause | Fix |
|---|---|---|---|
| 401 | `invalid_client` | `iss` or `sub` doesn't match the OAuth client UUID, or the signature failed to verify. | Recopy the UUID from the [Nabla Connect Admin](https://app.nabla.com/admin/nabla-connect); confirm your JWKS endpoint serves the public key referenced by `kid`. |
| 400 | `invalid_grant` | `aud` doesn't match the token URL for your region, or `iat`/`exp` falls outside the ±5-minute window. | Use the exact regional URL as `aud` and set `exp` ~5 minutes ahead. Sync your server clock via NTP. |
| 400 | `unsupported_grant_type` | `grant_type` is not `client_credentials`. | Set `grant_type` to `client_credentials`. |
| 400 | `invalid_request` | `client_assertion_type` is missing or incorrect, or required fields are missing. | Send all three body fields with the exact values shown above. |

For the full troubleshooting matrix, see [Authentication — Troubleshooting](/connect/authentication#troubleshooting).

<RequestExample>

```bash cURL
curl -X POST https://us.api.nabla.com/v1/connect/server/oauth/token \
  -H "content-type: application/json" \
  -d '{
    "grant_type": "client_credentials",
    "client_assertion_type": "urn:ietf:params:oauth:client-assertion-type:jwt-bearer",
    "client_assertion": "<YOUR_JWT_HERE>"
  }'
```

```js Node
const res = await fetch("https://us.api.nabla.com/v1/connect/server/oauth/token", {
  method: "POST",
  headers: { "content-type": "application/json" },
  body: JSON.stringify({
    grant_type: "client_credentials",
    client_assertion_type: "urn:ietf:params:oauth:client-assertion-type:jwt-bearer",
    client_assertion: assertion,
  }),
});
const { access_token, expires_in } = await res.json();
```

```python Python
resp = requests.post(
    "https://us.api.nabla.com/v1/connect/server/oauth/token",
    json={
        "grant_type": "client_credentials",
        "client_assertion_type": "urn:ietf:params:oauth:client-assertion-type:jwt-bearer",
        "client_assertion": assertion,
    },
).json()

access_token = resp["access_token"]
expires_in = resp["expires_in"]
```

</RequestExample>
